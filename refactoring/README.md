# Refactoring Overview

이 폴더는 SafetyCompanion 프로젝트의 구조 리팩터링을 계획하고 추적하기 위한 문서를 모아둔 곳이다.

이번 리팩터링의 목표는 기능 추가가 아니라 다음 세 가지를 우선적으로 해소하는 것이다.

1. CLI 진입점에 과도하게 몰린 오케스트레이션 책임 분리
2. 경로와 산출물 위치 정의의 분산 해소
3. 읽기 순서와 변경 영향 범위를 예측하기 쉬운 구조로 재정렬

문서 구성은 아래와 같다.

- `01_current_issues.md`: 현재 구조 문제와 리팩터링 필요성 정리
- `02_target_structure.md`: 목표 구조와 책임 분리 원칙
- `03_execution_plan.md`: 실제 적용 순서와 검증 기준
- `04_progress_log.md`: 이번 실행에서 적용한 변경 사항 기록

이번 턴에서 실제로 수행하는 리팩터링 범위는 다음과 같다.

- `src/main.py`를 슬림한 엔트리포인트로 축소
- CLI 명령 실행 코드를 별도 모듈로 분리
- 워크플로 오케스트레이션을 `src/workflows` 계층으로 이동
- 공통 경로와 인덱스 경로 정의를 `src/common/paths.py`로 중앙화
- 기존 테스트가 모두 유지되도록 회귀 검증 수행
